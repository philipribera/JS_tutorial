Declaring variables and the concept of 'undefined'

What is the difference between 'var' and 'let'? When do we use 'const'? Explain the concept of 'undefined' when it comes to variables Explain the difference between 'undefined' and 'null'.
Understanding 'undefined'
let a; // This will give 'a' the value undefined. // But we can also do this let a = undefined; // we declare 'a' and assign it the value 'undefined'
typeof(a) --> 'undefined'
a = "New value"; // we assign a new value to 'a'

// If we try to use a variable before being declared console.log(a); // Gives a reference error

Notes: 'undefined' is of datatype 'undefined' and a so called 'falsy' value.
Important: 'falsy' is not the same as 'false'. The latter being of datatype Boolean.
Undefined is not an error in it self, tough it's often the cause of errors/bugs. Javascript will automatically assign 'undefined' to a variable that are not initialized.

Typeof 'null'
let b = null;
typeof b --> object // null basically means empty object

Note: An easy way to check if an object is empty is to check the length of the 'keys' property in the Object constructor Note: If the purpose is to give a variable no/empty value it can be given the value 'null'. Primitive Types

What are primitive types? Understanding the above lets proceed Primitive types are immutable! But what does it really mean...?
let foo = "Hello World" foo.toUpperCase() --> 'HELLO WORLD' // we use a method on foo console.log(foo) --> 'Hello World' // but the original value is unchanged
// We can do this console.log(foo[0]) --> H // we get the 1st character // But not this... foo[0] = "T" --> "Hello World" // value doesn't change
Assigning a value to a variable gives it a new value, does not mutate the original value as with objects Noteworthy; in React 'props'(the object) can not be mutated, by design...
See more; https://developer.mozilla.org/en-US/docs/Glossary/Primitive More about objects passed by reference not by copy; https://stackoverflow.com/questions/16880418/javascript-pas
